## Suggested project workflow

1. Treat the registry as the source of truth. Papers should cite registry IDs internally during drafting, even if IDs are removed or hidden in final public manuscripts.

2. Every new proof/script should create or update one registry entry. No orphan scripts.

3. Every registry entry should eventually have: statement, status, source, dependencies, Nature target, caveat, next task, and paper projection.

4. Maintain separate layers: `results_master.yaml`, `open_targets.yaml`, `paper_projection.yaml`, and `artifact_inventory.csv`.

5. Update the dependency graph after every major registry change.
